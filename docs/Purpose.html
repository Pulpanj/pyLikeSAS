Purpose

Background

After more than 20 years of using the SAS System, when I started programming in Python I felt that I had lost many of the great features and capabilities that SAS provided.

SAS workloads typically read or update datasets and then produce output datasets or reports in various formats, while logging all details to the log file.

Replicating this workflow in my early Python code was not easy. I also struggled with changing Python versions and module dependencies, switching between development in Jupyter notebooks and scheduled batch jobs in plain Python scripts, and dealing with headaches related to output formats. And of course, the SAS data step was really missing in the pandas world.

These were the main motivations for creating the pyLikeSAS package.

And yes — the current pyLikeSAS implements a Pythonic version of SAS‑like datasets, the data step, ODS, and a few simple procedures.

What was the goal of the pyLikeSAS module

  • to have fun along the way
  • to implement SAS‑like formats and column labels on top of a DataFrame
  • to implement SAS dataset options on top of a DataFrame
  • to implement a function that processes a DataFrame in a SAS‑data‑step‑like manner
  • to implement the capability to generate various outputs (formats/styles) in a SAS ODS‑like fashion
  • to allow the pyLikeSAS package to be used both in Jupyter notebooks and as plain Python source files, with automatic switching of output modes
  • to gradually implement a few useful SAS PROCs as Python functions

What is (currently) not handled in the pyLikeSAS module

  • performance (it could be improved — or even tested…)
  • support for date/time columns (in the simple, limited, but elegant SAS style)
  • multiple input/output datasets in a single data step
  • user‑defined styles in PyODS
  • PyODS output formatting (currently printed as Python dicts — pretty‑printed, not JSON)
  • graphical output support using the terminalplot package
  • any kind of RTF format support
  • many more items not yet discovered…

A bit of history

pyLikeSAS is my toy project. It began as part of my Python learning and development journey during the Covid‑19 years, followed by several years of only sporadic progress in adopting Python. Now that I am retired, I finally have time to tinker with a project like this (which is definitely unusual in today’s AI‑driven world).

Over time, pyLikeSAS has evolved into a learning resource aimed at two distinct groups. pyLikeSAS can help to:

  • improve the Python skills of hard‑core SAS users (after all, a real programmer can write Fortran code in any language in any language), and

  • demonstrate the strengths of SAS to Pythonic “cake‑eaters” (For programmers born after 1990, a more readable version is here).

Hopefully, pyLikeSAS enables both groups to jump into Python‑based projects while staying confident that several things that work so well in SAS will remain fully under their control — and safely isolated from the ever‑changing versions of Python and its modules.

In a way, a tiny version of my dreamed‑of involvement in the development of the original SAS System did happen — just in Python 3.12 instead of Lattice C. I tapped into the vast ecosystem of Python open‑source modules, starting from a very different place than standard C libraries. And of course, I simply followed SAS functionality rather than reinventing all that great old SAS stuff myself.

I didn’t need to worry about performance or customers — it was all about my own learning and fun.

Disclaimer

In any case, the pyLikeSAS package is not intended for use in critical production code. That is why I am including this disclaimer (the same one I used in the old days, even on paid customer projects):

This software is not designed or intended for use in the on‑line control of aircraft, air traffic, aircraft navigation, or aircraft communications, or in the design, construction, operation, or maintenance of any nuclear facility.
