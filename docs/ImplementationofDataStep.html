Running Data Step with PdDataSet

Logic of the SAS Data Step

Ultimate goal of SAS data processing (apart from SQLs nad PROCs) is running data step. Data step is basically sequential processing of (rows of) input dataset during which variables (columns) are modified/appended with complex logic frequently based on the following data step rules:

  • every input row is processed independently (variables are read from input table). Order in which input rows are processed is defined by by parameter - that lists ordering variables.
  • the new derived variables could be defined in data step - but these has to be set within processing of one row in data step. Their state is reset for the next row of input.
  • only variables that keep their state during whole iteration over input rows are retained variables (variables defined only within data step - not read from input dataset). Retain variables should be defined as such and their initial value is usually set at the start of processing.
  • There is special (automatic) variable _n_ that is order of input row that is currently processed.
  • Similar to the (automatic) variable _n_ are (automatic) flag variables first._order_variable_ (and last._order_variable_) that indicates that particular ordering variable has the next value in the groups in sorted input rows (or has the last value in the groups). These variables allow to define complex group by processing only partially available in SQL window function.
  • row is (conditionally) written to output that could consist from subset of input variables/derived/retained variables.

Data step programming is very useful concept - when fully mastered it is hard to beat its simplicity,clarity and debugging enablement (when compared to SQL or DataFrame frameworks). Its implementation in pyLikeSAS is based on definition of python function that represent data step logic. That function is set as functional parameter of pyLikeSAS.data_step() function - that allows to define input/output data sets (and their dataset options), required ordering of input data set, retained variables, etc.

There should be two values returned by data step logic function, one is True/False flag indications that processed output row should be preserved/dropped. Current implementation supports just one output data set (and one input dataset)

  • that could be overcome by using sql parameter in the data set options of input data set.

(this is currently weakest point of pyLikeSAS - should be changed to allow more output rows from one call of data step logic function).

Simple Data Step example

Simplest example follows (note use of data set option on PdDataSet objects):

def simple_BMI(
        # here we list all input variables (read from input data set) 
        Name,  Age,  Height, Weight,
        # here we are adding retain variables (with initial values)
        ith_14plus,
        # here are all automatic variables (to be optionally in data step logic)
        _n_
):
    #    define conversion to cm/kg and set BMI for one row of class dataframe 
    #   + some game resembling SAS data step goodies
    # https://www.calculatorsoup.com/calculators/health/bmi-calculator.php


    Height = Height*2.54/100
    Weight = Weight*0.45359237
    # here we set value  new (potentially) output variables 
    # (defined in data step  logic)    
    # record input order into new variable
    nOBS = _n_+1  #SAS style nobs...
    BMI = Weight/(Height)**2
    # only cases older than 14 years are kept in output dataset
    if Age>14:
        # usage of retain variable
        ith_14plus += 1
        return True,locals()
    else:
        return 0, locals()

# read input df
df = PdDataSet(pd.read_csv("./data/class.csv").head(18))
df

dfout = data_step(
    df.ds_options(drop='sex'),
    data_step_code_function=simple_BMI,
    # keep list of variables to be passed to output dataset
    keep=[
        'nOBS', 'Name', 'Age',    'BMI','ith_14plus'
    ],
    retain={'ith_14plus':0},
    format={ 'ith_14plus': "{:.1f}"}
)

proc_print(dfout.ds_options(format={'BMI': "{:.1f}"}),template='seaside')

which of course generates output data set similar to SQL result shown in example of data set options. Its additional variable ith_14plus would be available in SQL example only after using some window functions (or aggregations).

More complete Data Step example

Second - more complex - data step example demonstrates usage of following features of data step (implementation in pyLikeSAS):

  • by processing including first/last flag variables
  • usage of retain variables
  • using native python object in the data step logic
  • usage of keep/label for output variables
"""
    test of pyLikeSAS.data_step using BMI index calculation
    and other bag of SAS tricks
"""


# function that converts BMI to message 
def bmi_label(value):
    # Function to categorize weight :
    if value < 18.5:
        return "Underweight"
    elif 18.5 <= value < 25.0:
        return "Normal"
    elif 25.0 <= value < 40:
        return "Overweight"
    else:
        return "Obese"


def inch_pounds(
        # data step automatic variables
        _n_, first, last, *,
        # input data set variables + retain variables
        Name,  Age, Sex, Height, Weight, sumBMI, BMI, nOBS
):
    """
        define conversion to cm/kg and set BMI for one row of class dataframe 
        + some game resembling SAS data step goodies
         _n_, first, last, * are comnpulsory "SAS automatic" variables
        the rest of var are
            -  columns of the input dataframe (in any order)
            - retain variables  (see retain parameter of data_step function)
    """
    nOBS = _n_+1  # SAS style nobs...
    Height = Height*2.54/100
    Weight = Weight*0.45359237
    BMI = Weight/(Height)**2
    msg = bmi_label(BMI)
    # skip row with Mary
    if Name == 'Mary':
        return 0, locals()

    Surname = surname_dict.get(Name, 'n/a')
    if Surname == 'n/a':
        surname_dict[Name] = 'Unknown aged:'+str(Age)
    if Sex == 'M':
        Name = 'Mr. '+Name
    else:
        Name = 'Miss '+Name

    #  first/last processing: note nice  SAS syntax and their immutability
    # print(type(first.Sex))
    if first.Sex:
        sumBMI = 0.
    sumBMI = sumBMI+BMI
    first_Sex = first.Sex
    # print(type(first_Sex))
    first_Age = first.Age
    last_Sex = last.Sex
    last_Age = last.Age
    return 1, locals()


# read input df
df = pd.read_csv("./data/class.csv")
df

# define objects to be used pythonically in data step logic inch_pounds
surname_dict = {'Alice': 'Caroll', 'James': 'Joyce'}

dfout = data_step(
    df,
    data_step_code_function=inch_pounds,
    retain={'BMI': 0, 'sumBMI': 0, 'nOBS': 0},
    # output dataframe columns order is defined by order in function or by keep
    keep=[
        'nOBS', 'Sex', 'Age',  'Name', 'Surname',  'BMI', 'sumBMI',
        'first_Sex', 'first_Age', 'last_Sex', 'last_Age'
    ],
    label={'Name': "Student's First Name", 'Sex': "Student Gender"},
    # or use drop=['Sex'],
    by=['Sex', 'Age'],
    by_order=[True, False]
)
proc_print(dfout)

SAS code for data step example

Above pyLikeSAS data step example could be written in SAS as follows:

/* SAS code for BMI */

proc format;
      value bmi_label
      /*    # Function to categorize weight :*/
      low - 18.5 = "Underweight"
      18.5 < - 25.0 = "Normal"
      25.0 < - 40 = "Overweight"
      other = "Obese"
;
run;


proc sort data = sashelp.class out = class;
      by sex descending age;
run;

data dfout;
      length name $50;
      retain BMI 0 sumBMI 0 nOBS;

      set class (obs=19 rename=(name=name_ori));
      by sex descending age;

      drop name_ori;

      nOBS = _n_;
      Height = Height*2.54/100;
      Weight = Weight*0.45359237;
      BMI = Weight/(Height)**2;

      msg = put(BMI,bmi_label.);

      /*     skip row with Mary*/
      if name_ori='Mary' then do;
            return;
      end;

      /*    surname dictionary could be implemented using hash...*/
      /*    Surname = surname_dict.get(Name, 'n/a')*/
      /*    if Surname == 'n/a':*/
      /*        surname_dict[Name] = 'Unknown aged:'+str(Age)*/
      if Sex = 'M' then
          Name = 'Mr. ' || name_ori;
      else
          Name = 'Miss ' || name_ori;

      if first.Sex then do;
          sumBMI = 0.;
      end;
      sumBMI = sumBMI+BMI;
      first_Sex = first.Sex;
      first_Age = first.Age;
      last_Sex = last.Sex;
      last_Age = last.Age;
      output;
run;
proc print;run;

SAS code is remarkably similar - just update of surname dictionary was not implemented (could be done using SAS hash object).
