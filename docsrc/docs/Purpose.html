Purpose

Background

pyLikeSAS is my toy project. It started as part of my python learning and development path (back in Covid-19 years, followed by years of sporadic progress in my python adoption).

And now … yes, currently implemented pyLikeSAS is pythonic version of SAS-like dataset, data step, ODS and a few simple procedures.

So a tiny version of my dreamed involvement in development of the original SAS System did happened - just in python v3.12 instead of Lattice C. I did tap into the vast ecosystem of python open source modules, so starting in the way different place than a standard C libraries. Also I just followed SAS functionality and not invented all that old great SAS stuff. I did not need to care about performance and customers - it was all about my own learning and fun.

Now it seems, pyLikeSAS turned into learning resource aimed to be consumed by two distinct groups. pyLikeSAS could help to:

  • improve python skills of hard core SAS users (as the real programer can write Fortran code in any language ) and
  • demonstrate strengths of SAS to pythonic cake-eaters https://pbm.com/~lindahl/real.programmers.html

Hopefully, it enables both these groups to jump into projects coded in python while being confident that couple of things so well working in SAS will be fully in their hands and feel safely isolated from the ever changing python and its modules versions.

My 20+ years experience with SAS workloads, that basically read/update data and generate outputs in various formats while logging detail log file was not easily replicated in my beginner python. It was the main motivation for PyODS and PLSlogger. In additions such python workloads are developed in Jupyter and later scheduled to be run in batch (i.e. like in terminal). So pyLikeSAS package is designed to be used in Jupyter notebooks or as the plain python sources with automatic switching of outputs. And of course SAS data step was really missing in the pandas world.

In any case pyLikeSAS package is not intended to be used in the production code. That is why I am placing here the disclaimer (that I had been using in the old times even on the paid customer projects):

This software is not designed or intended for use in on-line control of aircraft, air traffic, aircraft navigation or aircraft communications or in the design, construction, operation or maintenance of any nuclear facility.

What was goal of pyLikeSAS module

  • implement SAS data set like formats and labels on columns of the DataFrame
  • implement SAS data set options on top of DataFrame
  • implement function that will process DataFrame like SAS data step
  • implement capability to generate various outputs (formats/styles) in SAS ODS like fashion
  • gradually implement couple of useful SAS PROCs as python functions
  • having fun all the way along

What is (now) not handled in pyLikeSAS module

  • performance, could be improved (or even tested)…
  • support for date/time columns (in simple, deficient, but elegant SAS manner)
  • more input/output data sets in the data step
  • definition of user styles in PyODS
  • PyODS prints in python dict format (prettyprint not json)
  • support of graphical outputs using terminalplot package
  • any kind of rtf format support
  • many more items still not spotted yet …
